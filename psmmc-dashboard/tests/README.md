# PSMMC dashboard — regression specs

Browser specs that make calculation regressions impossible to ship quietly.
They drive the real `index.html` from `file://` via playwright-core chromium.

## Run

```bash
node psmmc-dashboard/tests/run.mjs        # regenerates fixtures, runs every spec
node psmmc-dashboard/tests/spec-sample.mjs # any spec is standalone (exit != 0 on fail)
```

## Env vars

- `PSMMC_CHROME` — chromium binary (default `/opt/pw-browsers/chromium-1194/chrome-linux/chrome`).
- `PSMMC_PW_DIR` — dir to resolve `playwright-core` from (default `/tmp/pwtest`).

## What each spec protects

- **spec-sample** — sample data loads with zero page errors; the 4 Planning decision figures (283 / 72 / 109.7M / ≈9.5M + MoM badge); order sheet = 7 rows; Management value card shows **no negative figure** (audit A1, the "−2432M" bug); Averages renders.
- **spec-upload** — detected period chip (01 Jan → 31 Mar); in-stock code avg/coverage/status from `fixtures/expected.json`; non-drug (prefix-4) and REJECTED rows excluded (row count = 2); 6-month override rescales avg = total ÷ 6 and marks months_source=manual.
- **spec-calendar** — calendar invariant (audit A2): min date 2026-06-01 shows "01 Jun" not "31 May" under both Asia/Riyadh and America/New_York (no `toISOString()` day-shift).
- **spec-dedup** — guards audit B1: same-content withdrawals uploaded twice must NOT double totals/averages.
- **spec-xss** — guards audit A4: a `5<img onerror=…>` NUPCO code never executes (window.__xss undefined, no live img).

Fixtures are generated by `make-fixtures.mjs` with the vendored SheetJS; delivery dates are ISO-string cells (see that file's header for why).
