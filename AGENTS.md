EDDAH / عدة — Agent Instructions

Project scope

This repository contains the EDDAH / عدة landing page project.

Primary working folder:
eddah-landing

Do not create a new project unless explicitly requested.

Brand

EDDAH / عدة is a premium local-first home maintenance service in Riyadh.

Current service area:
حي لبن فقط

Language:
Arabic-first RTL

Conversion:
WhatsApp-first

Services scope — strict

Only these three services are allowed:

1. السباكة
2. الكهرباء
3. التكييف

Do not add, mention, imply, or create sections for:

* water tank cleaning
* painting
* carpentry
* cleaning
* pest control
* appliance repair
* general maintenance categories
* any other service

Every card, icon, image, FAQ, CTA, and copy must stay within:
السباكة، الكهرباء، التكييف فقط.

Visual identity

Use the real uploaded images in:

eddah-landing/public/brand

Do not use:

* stock photos
* placeholders
* CSS-only visuals instead of real images
* old dark assets
* random generic service photos

The حي لبن map image is the main color and local-identity source.

Expected map file:
eddah-landing/public/brand/laban-map.png

The map image should show:

* orange pin
* label “حي لبن”
* label “غرب الرياض”
* EDDAH / عدة logo
* light beige map background

Use this map as:

1. The main حي لبن visual section
2. The color source for the website

Color system

Derive the site palette from the map image.

Use:

* warm off-white
* ivory
* soft beige / sand
* light stone
* warm light gray
* amber/orange from the map pin
* subtle charcoal for text
* optional copper/bronze details

Avoid:

* black-heavy sections
* dark cinematic backgrounds
* neon colors
* excessive orange
* harsh contrast
* generic SaaS gradients

Quality reference

Use this reference only for quality, composition, motion rhythm, and premium layout sophistication:

https://orbai-template.framer.website/

Do not copy:

* assets
* code
* content
* colors
* identity

The goal is not to clone Orbai.
The goal is to reach a similar level of:

* advanced cards
* layered panels
* image placement
* motion rhythm
* spacing
* mobile polish
* premium front-end feel

Design standards

The site must not look like basic rectangular service cards.

Use:

* layered image compositions
* overlapping panels
* premium editorial crops
* floating badges
* glass/stone panels
* warm surfaces
* subtle map-line accents
* soft shadows
* asymmetrical layouts when useful
* strong mobile-first composition

Motion standards

Use the existing stack first.
Do not add heavy motion libraries unless necessary.

Motion should be:

* subtle
* premium
* smooth
* purposeful
* mobile-friendly
* reduced-motion friendly

Good patterns:

* hero entrance
* staggered reveal
* hover/tap depth
* image mask/reveal
* sticky nav transition
* soft section reveal

Avoid:

* bouncing
* random movement
* excessive parallax
* performance-heavy effects

Mobile standards

Mobile is critical.

On iPhone:

* hero must be clear in the first screen
* CTA should be visible early
* service cards must not become boring tall rectangles
* image crops must stay intentional
* spacing must not feel cramped
* nav must be usable
* Arabic text must be readable and elegant
