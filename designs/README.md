## كيف يختار impeccable المشروع الفعّال

`context.mjs` يقرأ سياق **مشروع واحد** في كل مرة، بهذا الترتيب:

1. `PRODUCT.md`/`DESIGN.md` في مجلد التشغيل (الجذر) — **متروك فاضي عمدًا** عشان ما يقفل على مشروع واحد.
2. `.agents/context/` ثم `docs/` — غير مستخدمة.
3. **`IMPECCABLE_CONTEXT_DIR`** — هنا نحدّد المشروع. مثال:
   ```bash
   export IMPECCABLE_CONTEXT_DIR=designs/eddah   # اشتغل على عدّة
   ```
4. وإلا → لا سياق → impeccable يبدأ عامًّا ويسألك أي مشروع.

**القاعدة:** قبل أي أمر تصميم لمشروع معيّن، حدّد `IMPECCABLE_CONTEXT_DIR=designs/<المشروع>`.
بدونها، impeccable يطبّق الأساس العام أدناه ويسأل.

## إضافة مشروع جديد

```bash
cp -r designs/_template designs/<اسم-المشروع>
# عبّئ PRODUCT.md (وش المنتج + register) و DESIGN.md (الهوية فوق الأساس)
export IMPECCABLE_CONTEXT_DIR=designs/<اسم-المشروع>
# /impeccable craft <الميزة>
```
اختر `register` في PRODUCT.md حسب النوع:
- **`marketing`** → موقع هبوط/حملة (التصميم *هو* المنتج) — يقرأ impeccable `reference/brand.md`.
- **`product`** → Dashboard/أداة/نظام تشغيل (التصميم *يخدم* المنتج) — يقرأ `reference/product.md`.

# الأساس العام (House Baseline)

قواعد البيت — تنطبق على **كل** مشروع. كل `DESIGN.md` لمشروع يبدأ من هنا ويضيف هويته فوقه.

## الاتجاه واللغة
- **عربي أولاً، RTL كامل** (`dir="rtl"`، `lang="ar"`). صمّم للعربية أصالةً لا كترجمة.
- **أرقام عربية** (٠ ١ ٢ ٣) حيث تناسب الهوية.
- **لا تتبّع (letter-spacing) سالب قوي على العربية** — الحروف متّصلة؛ `-0.01em` كحدّ أقصى.

## الخط
- **خط البيت الافتراضي: خط ثمانية (Thmanyah)** — Sans للنصوص، Serif Display للعناوين.
  الملفات الجاهزة (woff2 + otf + الترخيص) في [`eddah/fonts/`](eddah/fonts/)؛ أعد استخدامها
  في أي مشروع ما لم يفرض المشروع خطًّا آخر.
- سقف عائلات الخط: ٣ كحدّ أقصى. هرمية عبر الوزن + المقياس (تباين ≥ 1.25 بين الدرجات).

## اللون والتباين
- نص body ≥ **4.5:1** على خلفيته؛ النص الكبير ≥ 3:1؛ placeholder بنفس 4.5:1.
- لا رمادي خافت «للأناقة» على خلفية فاتحة مائلة — أكبر سبب لصعوبة القراءة.
- اختر **استراتيجية لون** قبل الألوان (مقيّدة / ملتزمة / لوحة كاملة / مُغرِقة)، وثبّت عليها.

## الحركة
- منحنى ease-out أسّي (لا ارتداد، لا elastic). لغة عدّة: `cubic-bezier(0.16, 1, 0.3, 1)`.
- كل حركة لها بديل في `@media (prefers-reduced-motion: reduce)` — غير اختياري.
- لا تُخفِ المحتوى خلف كلاس حركة (الكشف يفشل في التبويبات المخفية).

## الإتاحة والمتانة
- لمسة ≥ 44px · تركيز واضح (`:focus-visible`) · ترتيب DOM منطقي · بدائل نصية للصور.
- حالات: فاضي / تحميل / خطأ / طويل (overflow) — صمّمها، لا تفترض المثالي.

## المجسمات ثلاثية الأبعاد (3D)
- المرجع الكامل: [`references/3d-objects.md`](references/3d-objects.md).
- المصدر الأساسي: [thiings.co](https://www.thiings.co/things) (+١٠٠٠٠ مجسم PNG شفاف). المجاني **شخصي + نسبة**؛ التجاري يحتاج خطة مدفوعة.
- بديل تجاري مجاني: [3dicons.co](https://3dicons.co) (مفتوح المصدر، بلا نسبة).
- التوظيف: مجسم بطل واحد لكل قسم/حالة فاضية، أسلوب موحّد، ظل عدّة الناعم، يبان في الوضعين، احفظه في `assets/3d/`.

## ممنوعات (AI slop)
- حدود جانبية ملوّنة (side-stripe) · نص متدرّج (`background-clip:text`) · زجاج كافتراضي ·
  قالب «الرقم الضخم» · شبكات بطاقات متطابقة · eyebrow صغير مكرّر فوق كل قسم ·
  ترقيم أقسام افتراضي (01/02/03) بلا تسلسل حقيقي · نص يفيض حاويته.
- نسخة: كل كلمة تستحق مكانها · **لا em dash** · لا buzzwords (streamline/empower/seamless…) ·
  أزرار «فعل + مفعول» («اطلب الفني» لا «موافق»).
