# Premium Framer Landing Polish Skill

## Goal

Transform a basic landing page into a premium, polished, non-generic interface with strong visual hierarchy, refined spacing, layered composition, intentional image placement, and tasteful motion.

## When to use

Use this skill when the user asks for:

* premium Framer-like quality
* Orbai-level sophistication
* high-end landing page polish
* better cards
* better motion
* better mobile design
* non-generic service website design

## Core principles

* Do not create basic rectangular card grids.
* Avoid generic SaaS/service-card layouts.
* Avoid stock-template feeling.
* Use layered compositions, image masks, overlapping panels, floating badges, soft shadows, glass, stone/light surfaces, and asymmetry.
* Maintain the user’s brand identity, real images, colors, and service scope.
* Use reference sites only for quality and composition principles, never for copying.

## Brand preservation

For EDDAH / عدة:

* Keep Arabic RTL.
* Keep the brand light, warm, premium, and local.
* Keep the حي لبن map image as the color and locality anchor.
* Keep only three services:
    * السباكة
    * الكهرباء
    * التكييف
* Use the real brand images from /public/brand.
* Do not introduce unrelated services.

## Design checklist

Before finishing, verify:

1. Hero has strong composition, not just text + image.
2. Cards have depth, hierarchy, and custom structure.
3. Images are cropped intentionally.
4. Mobile layout feels designed, not squeezed.
5. Spacing is generous and consistent.
6. Typography has clear hierarchy.
7. CTAs are obvious and premium.
8. Motion is subtle and purposeful.
9. Page does not feel generic or AI-generated.
10. Brand assets are used intentionally.

## Card design patterns

Prefer:

* layered cards
* card-within-card structure
* editorial image crops
* overlapping content panels
* floating badges
* glass panels
* warm stone/ivory surfaces
* subtle orange accent lines
* map-line decorative details
* responsive asymmetric grids

Avoid:

* equal-height boring boxes
* flat image + text cards
* heavy borders
* generic icon cards
* repetitive layouts

## Motion checklist

Use existing project animation tools first.
Prefer CSS or existing motion utilities unless the project already has Framer Motion or another library.

Motion should include:

* hero entrance
* staggered reveal
* subtle hover depth
* image reveal/mask
* smooth section transitions
* reduced-motion support

Avoid:

* excessive bouncing
* random movement
* heavy performance effects
* animation that hurts mobile usability

## Mobile checklist

On mobile:

* hero must be clear within first screen
* CTA must be visible early
* cards must not become boring tall rectangles
* image crops must remain intentional
* spacing must not feel cramped
* nav must be usable
* text must remain readable
* the map section must remain visually strong
